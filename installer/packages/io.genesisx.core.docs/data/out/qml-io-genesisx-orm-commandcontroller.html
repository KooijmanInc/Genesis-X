Detailed Description

Signal Documentation

requestFinished(string path, bool ok, int status, string error, object obj)

Emitted when the postrequest got response. The path where the request was send to. The ok boolean if it was successful. A http response status e.g. 200 or 400. The error shows what it wrong when getting other than status 200 The return body that must be in an JsonObject obj.

Note: The corresponding handler is onRequestFinished.


Method Documentation

void cmdLogin(string user, string pass)

Set login credentials for JWT authorization, add user as username and pass as password


void cmdPing()

Returns a signal if connection to remote server is successful


cmdPostJson(string path, var vm)

Send path as the path in url for connecting to API function Send vm as the QVariantMap as parameters you need at that url request e.g. Customernumber 1, Address Krijtlaan etc.


cmdPostJsonBoolAsync(string path, object body)

Send path as the path in url for connecting to API function Send body as a QJsonObject, it returns only a boolean if post was successful


cmdPostJsonBoolAsync(string path, object body)

Send path as the path in url for connecting to API function Send body as a QJsonObject. Its the same as cmdPostJsonBoolAsync but it returns HttpResponse with body


cmdRegister(string email, string password)

A Stub to check if postJson is correct, testing the connection Send email and password for testing
