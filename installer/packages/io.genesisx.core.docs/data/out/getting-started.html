Getting started with Genesis-X

Overview

This guide walks you through installing Genesis-X, enabling the modules in your Qt project, and integrating the documentation and tools into Qt Creator.

By the end of this page you will have:

  • Genesis-X installed on your system.
  • Documentation registered in Qt Creator / Qt Assistant.
  • A minimal project using the Genesis-X Core module.
  • Optional: QML assets and ORM enabled.

Requirements

  • Qt 6.5 or later (Genesis-X is tested with Qt 6.9 / 6.10).
  • Qt Creator (recommended for best integration).
  • A supported compiler toolchain per your Qt version.
  • CMake or qmake; Genesis-X focuses on qmake integration.

Installation

Genesis-X is distributed with an installer script:

  • install.sh for Linux and macOS.
  • install.bat for Windows (or install.sh in a bash environment such as Git Bash).

Run the installer from a terminal:

# Linux / macOS
chmod +x install.sh
./install.sh

On Windows (CMD or PowerShell):

install.bat

The installer guides you through:

  • Installing Genesis-X libraries and headers.
  • Installing QDoc / Qt help files.
  • Registering documentation in Qt Creator.
  • Installing Qt Creator snippets.
  • Installing Qt Creator wizard templates.
  • Optionally installing third-party integrations (for example, Firebase support).

Qt Creator integration

If you chose to install documentation and tools, the installer automatically updates Qt Creator configuration so that:

  • Genesis-X documentation appears under Help > Contents.
  • The search index includes Genesis-X types and topics.
  • Qt Creator snippets become available for QML and C++.
  • Wizard templates appear in File > New Project.

If needed, you can manually add the documentation by opening:

  • Tools > Options > Help > Documentation
  • Click Add...
  • Select the genesisx.qch file installed by Genesis-X.

Adding Genesis-X to a qmake project

To use Genesis-X in an existing qmake-based project, open the .pro file and add the required modules.

At minimum, add the Core module:

QT += genesisx

Enable additional modules as needed:

QT += genesisx-utils      # helpers and utilities
QT += genesisx-orm        # ORM and HTTP controllers (optional)
QT += genesisx-physics    # physics for View3D (optional)

Then run qmake again (or trigger a reconfigure in Qt Creator) and rebuild the project.

Using the QML design system (assets)

The Genesis-X design system is shipped as a QML module, typically under the name assets.

In your QML files:

import QtQuick
import QtQuick.Controls
import assets 1.0

ApplicationWindow {
    width: 800
    height: 600
    visible: true

    // Example usage of a design-system component
    Column {
        anchors.centerIn: parent

        AppTitle {
            text: "Hello, Genesis-X"
        }

        PrimaryButton {
            text: "Continue"
        }
    }
}

The exact components available depend on the version of the assets module shipped with your Genesis-X distribution. Refer to the Assets Module documentation for the full API.

Minimal C++ setup example

The following example shows a minimal C++ main.cpp that uses Genesis-X alongside Qt Quick:

#include <QGuiApplication>
#include <QQmlApplicationEngine>

#include <GenesisX/Core/GenesisXApplication.h>

int main(int argc, char *argv[])
{
    QGuiApplication app(argc, argv);

    gx::core::GenesisXApplication gxApp;
    gxApp.initialize();

    QQmlApplicationEngine engine;
    const QUrl url(u"qrc:/main.qml"_qs);

    QObject::connect(&engine, &QQmlApplicationEngine::objectCreated,
                     &app, [url](QObject *obj, const QUrl &objUrl) {
        if (!obj && url == objUrl)
            QCoreApplication::exit(-1);
    }, Qt::QueuedConnection);

    engine.load(url);

    return app.exec();
}

The exact C++ entry classes (for example GenesisXApplication) may vary depending on your installed Genesis-X version and chosen modules. Check the Core Module documentation for the authoritative API reference.

Creating a new project using the wizard

When the wizard templates are installed, you can create a new Genesis-X project directly from Qt Creator:

  1. Open Qt Creator.
  2. Select File > New Project.
  3. In the left-hand Projects tree, locate the Genesis-X category.
  4. Choose a Genesis-X application template (for example, Genesis-X Quick Application).
  5. Follow the wizard steps to configure the kit, build location, and project options.

The generated project already contains the correct QT += genesisx entries and a minimal QML / C++ setup.
